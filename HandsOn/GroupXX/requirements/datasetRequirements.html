Dataset Requirements for Waste Collection in Madrid

Overview

The dataset selected contains information on waste collection in Madrid, including the type of waste, the district where it is collected, and the amount of waste collected over time.

Requirement Evaluation

R1: Data are in one of the selected Smart City domains

Satisfied: The dataset falls within the "Waste Management" domain, which is a key area of Smart City development. The data relates to waste collection by district and type of waste.

R2: Data are available as a CSV file

Satisfied: The dataset is available in CSV format.

R3: Data have an open license

The data can be reused for commercial and non-commercial purposes under an open license.

R4: Data can be easily linked with generic real-world entities

Satisfied: The dataset includes geographical entities such as districts of Madrid, which can easily be linked to real-world entities using geographical datasets like DBpedia or GeoNames.

R5: Documentation exists for the data (Optional)

Not Satisfied: The dataset does not come with detailed documentation. However, the column names are intuitive, and the structure is easy to interpret.

R6: Data can come from multiple data sources (Optional)

Not Satisfied: This dataset is provided by a single source, the waste collection system of Madrid. However, it could be enriched by combining it with other datasets (e.g., demographic or geographic data).

Linking to Other Datasets

The dataset includes district-level information, specifically the district names and codes, which can be linked to external datasets. For example, the districts of Madrid can be linked to demographic datasets that provide additional data such as population, income levels, or socio-economic indicators for each district. Additionally, the dataset could be linked with geographic datasets, like those found in DBpedia or GeoNames, to provide geographical context and further enhance the data's usefulness.
