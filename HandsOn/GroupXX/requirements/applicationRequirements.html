Application Requirements for Waste Collection Visualization

Functional Requirements

1. Waste Collection Map

The application must display a map of Madrid showing all the waste collection points. Each point should provide information on the type of waste collected (e.g., "Cardboard", "Glass") and the quantity collected.

2. Filter by District and Waste Type

The user should be able to filter the waste collection points by district and by type of waste, enabling them to view relevant information according to their needs.

3. Data Visualization

The application should provide a graphical visualization (e.g., bar charts) of the amount of waste collected by month and type of waste.

Non-Functional Requirements

1. Usability

The application should be easy to use and intuitive. The user should be able to quickly access information with minimal clicks.

2. Performance

The application should load and display data in less than 3 seconds under normal load conditions.

3. Accessibility

The application should follow standard web accessibility guidelines to ensure it is usable by a wide range of users, including those with disabilities.
